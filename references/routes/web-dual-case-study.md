# Case Study Template

Surface: `web_dual`
Document Shape: `case_study`

## Fixed section recipe
1. Context
2. Problem
3. Approach
4. Evidence
5. Outcome
6. Takeaway

## Layout notes
- lead with enough context to make the outcome meaningful
- keep the sequence explicit; do not shuffle evidence ahead of the problem
- charts should support the narrative, not replace it
- the page should remain readable in print/PDF

## Example scaffold
```yaml
context:
  components: [eyebrow, title_block, summary_strip]
problem:
  components: [body_paragraph, fact_card]
approach:
  components: [body_paragraph, fact_card]
evidence:
  components: [chart_frame, insight_callout]
outcome:
  components: [fact_card, body_paragraph]
takeaway:
  components: [method_note, footer_meta]
```
