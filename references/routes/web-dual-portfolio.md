# Portfolio Route Reference

Surface: `web_dual`
Document Shape: `portfolio`
Public document type: `portfolio`

## Fixed section recipe
1. Profile or context
2. Selection overview
3. Case blocks
4. Process or capability
5. Closing meta

## Layout notes
- use the route to curate multiple pieces of work
- keep each case block comparable
- diagrams are allowed when they clarify process, not as decoration

## Example scaffold
```yaml
profile_or_context:
  components: [eyebrow, title_block, summary_strip]
case_blocks:
  components: [fact_card, body_paragraph]
process_or_capability:
  components: [chart_frame, insight_callout]
closing_meta:
  components: [footer_meta]
```
