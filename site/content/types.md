引擎本身是通用渲染器：任意 `document_type` 都可使用全部组件，引擎不按类型做结构校验。`document_type` 是 frontmatter 中的语义标签，出现在页头并引导 Skill 选择模板。以下 15 份样板由示例画廊与 Skill 引导提供，每份都有约定的章节结构、排版层级与中英文版本，作为编写各类文档的起点。

## one_pager · 方案 / 执行摘要

在 1-2 页内扼要呈现核心观点。适合商业方案、项目提案、执行摘要。四段结构：摘要 → 背景 → 证据 → 建议，每段可含正文、要点列表与数据卡片。

## long_doc · 白皮书 / 长文

撰写深度长文，含多个章节与引用。适合白皮书、技术文档、分析报告。摘要 + 可迭代章节（每章含标题、正文、可选引用块）+ 结论。

## letter · 信件 / 推荐信 / Memo

正式信函、推荐信、内部备忘录。信头（收件人 / 发件人 / 日期）→ 正文段落 → 签名区。

## portfolio · 作品集

展示项目案例或工作成果。概览 → 案例列表（每项含标题 / 背景 / 行动 / 结果 / 指标卡）→ 能力总结。

## resume · 简历 / CV

求职或个人简介。个人信息 → 工作经历（含时间线）→ 教育背景 → 技能标签 → 项目列表。打印友好。

## slides · 幻灯片 / 演示

PPT 风格演示文稿。适合汇报、路演、教学。16:9 视口，支持 cover、closing 等多种布局。

## equity_report · 研报 / 估值分析

金融研究与估值报告。投资论点 → KPI 数据卡 → 趋势图（含 K 线）→ 估值 → 风险矩阵 → 催化剂时间线 → 免责声明。

## changelog · 更新日志 / Release Notes

版本更新说明。版本列表，每版含日期 / 版本号与新增 / 改进 / 修复三类变更，各类以颜色与标签区分。

## process_flow · 流程 / SOP

操作流程与标准作业程序。步骤列表（每步含标题 / 描述 / 负责人）+ 可选泳道图。

## timeline · 时间线 / Roadmap

时间线与路线图。每项含日期 / 标题 / 描述，支持 completed / in_progress / upcoming 三种状态，状态以颜色与标签区分。

## faq_page · 常见问题

结构化 FAQ。引导语 → 分组问答（每组含标题与问答列表）→ 总结。

## case_study · 案例拆解 / 项目复盘

深度拆解一个项目。背景 → 挑战列表 → 解决方案（含步骤）→ 结果指标 → 经验教训。

## research_summary · 研究摘要

快速呈现研究要点。研究问题 → 关键发现（编号列表）→ 方法说明 → 局限性 → 下一步。

## stats_report · 数据报告 / KPI

数据驱动的分析报告。KPI 指标卡 → 趋势图（可嵌入图表）→ 数据分群 → 方法说明。

## infographic · 信息图

将复杂信息视觉化。适合年报摘要、数据故事。大数字 + 图表 + 要点列表的组合呈现。
