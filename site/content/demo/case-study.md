## 背景

某金融科技公司（以下简称 F 公司）拥有 800 名员工，其中分析团队 40 人，每月产出 60+ 份各类报告——包括行业研报、个股分析、风险月报和监管合规文件。所有报告都依靠分析师手工排版（Word + PDF），格式不一致、更新成本高、难以自动化。

F 公司 CIO 提出了明确目标：将报告产出周期从 3 天缩短至 2 小时，同时确保所有报告视觉一致、符合品牌规范。

## 问题

经过两周的深入调研，我们识别出三个核心痛点：

:::kpi-row
items:
  - label: 月均报告量
    value: 60+ 份
  - label: 平均排版时间
    value: 4.5 小时/份
  - label: 格式不一致率
    value: 65%
  - label: 自动化率
    value: 0%
:::

## 方案

我们采用分阶段迁移策略，避免一次性推倒重来：

:::steps
title: 分阶段迁移
items:
  - step: 1
    title: Phase 1 · 替换高频类型
    desc: 用 equity_report 和 stats_report 类型替换最高频的两种报告（研报、风险月报），覆盖月报量的 60%。
  - step: 2
    title: Phase 2 · 半自动化
    desc: 分析师定义数据模板，脚本从数据库自动提取数据填入模板，实现半自动化。
  - step: 3
    title: Phase 3 · 平台集成
    desc: 将引擎集成到内部数据平台，分析师在 Web UI 中选择报告类型和参数，后台自动生成并渲染 HTML。
:::

## 结果

迁移上线 3 个月后的效果数据：

:::kpi-row
items:
  - label: 报告产出时间
    value: 0.5 小时
    note: 从 4.5 小时缩减 89%
  - label: 月均报告量
    value: 85+ 份
    note: +42%
  - label: 格式一致性
    value: 100%
    note: 设计令牌保证零偏差
  - label: 分析师满意度
    value: 4.6/5
    note: 从重复劳动中解放
:::

项目最大的意外收获是激发了分析团队的创造力。当排版不再是瓶颈后，分析师开始主动尝试新的报告形式——从传统的研报扩展到了信息图、时间线和案例拆解。这在旧流程中几乎不可行，因为每种新格式都意味着数小时的手工 CSS 调试。

> Momo Paper 节省的不仅是排版时间，更是整个组织对「报告可以长什么样」的想象力。
> — F 公司 CIO

## 经验教训

这个案例揭示了设计系统在企业中的一个被低估的价值：它不仅统一了视觉语言，更降低了「创造新格式」的边际成本。当从想法到成品的时间从 3 天缩短至 30 分钟，组织的信息表达能力发生质的飞跃。

- 分阶段迁移比一次性替换更稳妥，先用高频类型建立信任
- 结构化 DSL 作为中间格式，让分析师和工程师各司其职
- 打印安全和屏幕预览的一致性在金融合规场景中至关重要
- 设计令牌的「约束」反而是解放——分析师不再纠结于排版选择
