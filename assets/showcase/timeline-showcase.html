Momo Paper / web_dual.timeline

时间线最适合把“过程感”讲得有秩序。

当内容按阶段发生时，时间线比普通列表更有解释力。它天然适合 roadmap、项目推进、产品演进、事件脉络，甚至也能成为简历和案例故事的骨架。

Best Use
Roadmap / Milestone
适合连续事件、阶段计划、产品演变、任务推进和案例复盘。

一个从规范到落地的演进示例

Design system adoption
Phase 01 / Define

建立统一 token 与 route 规则

先收紧颜色、字体、间距、图表规则和组件白名单，避免后续每次生成都重新决定风格。

Phase 02 / Connect

把双轴 route 接进 agent 调用链

让每次任务先解析 `surface + document_shape`，再读取 machine-readable tokens 与 prompt contract。

Phase 03 / Expand

扩展到更多结构化内容

在 `web_dual.explainer` 之外，逐步补齐 `process_flow`、`timeline`、`resume_profile`、`faq_page`、`case_study` 与 `research_summary`。

Phase 04 / Audit

建立 route 与 print-safe 校验

对颜色、字体、图表、标题层级、语气与打印可读性进行自检，减少产物漂移和风格失真。
