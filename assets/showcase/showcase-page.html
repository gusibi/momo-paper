Momo Paper / web_dual.explainer

结论优先，然后才是风格。

这是一个按新设计系统落地的单页示范。它不追求炫技，而是用统一的色彩、版式、文字气质与数据表达，把网页、PPT、信息图和报告的共同视觉基因压到同一条轨道上。

Shared DNA / Variant by Medium

这套系统的核心不是模板套模板，而是先把不该变化的部分收紧。颜色只来自固定 token，标题和正文各自有明确的字体职责，图表只允许使用少数几种结构，组件也按媒介白名单管理。

在这个前提下，不同媒介再去做自己的事。网页强调滚动节奏，PPT 强调单页结论，信息图强调路径引导，文章强调阅读舒适度，数据报告强调方法与证据透明。家族相似，但不是机械复制。

01 / Type

标题是判断器

标题必须先交代主题或结论，不能拿情绪口号代替结构。

02 / Color

色彩是编码，不是装饰

主色负责结构，强调色负责提醒，语义色只在语义场景出现。

03 / Density

密度按任务切换

同一品牌下允许 calm、standard、dense 三档，但不能随意失控。

“先让 agent 定 route，再让它写内容。”

如果系统只是一份文档，agent 依旧会在每次生成时临场发挥。真正稳定的做法，是让 token、route registry、prompt contract 和 checklist 一起成为生产链的一部分。

Consistency Trend
系统化输出一致性 无规范时的漂移风险

Key Takeaway

当风格约束进入生成流程后，一致性会上升得很快。

真正起作用的不是“设计说明”，而是结构化约束是否进入了调用链。只要 agent 仍然能自由换色、换字、换模块，结果就会继续乱飘。

Method Note

本页示范使用 Momo Paper 的 `web_dual.explainer` route，采用 `standard` 密度、serif 标题、sans 正文、单主色 + 单强调色，并保持浏览器阅读与 print-safe 文档秩序。

这页不是模板本身，而是模板应该长出的样子。

如果你认可这个方向，下一步就应该把这套系统继续接到你的 agent 工作流里，让页面、PPT、文章和图表都按同一条规则生成，而不是每次重新“设计一次”。
