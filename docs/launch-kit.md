# Momo Paper Launch Kit

这是推广执行入口，不再重复保存多套计划和文案。

## 文件导航

- [`growth-plan.md`](./growth-plan.md)：ICP、首发定位、渠道分工、短板、指标和决策规则。
- [`content-calendar.md`](./content-calendar.md)：30 天逐日执行日历与发布台账。
- [`promotion-copy.md`](./promotion-copy.md)：中英文长文、社区帖、短帖、轮播、私信和评论回复。
- [`benchmark-protocol.md`](./benchmark-protocol.md)：完成量化传播前必须遵循的对照实验方法。

## 首发最小资产

发布前只要求以下内容完成：

1. 一个可公开、可复现的 equity report 英雄案例。
2. 一个 30–45 秒演示 GIF 或视频。
3. 一台干净环境成功走完 README Quick Start。
4. GitHub 承接页、一个中文技术社区主帖和 3 位一对一试用对象。

Benchmark 不是第一轮发布门槛。没有完成受控实验时，只能定性描述“把重复布局与 CSS 移出 Agent 输出”，不能发布具体节省比例。

## 第一周发布顺序

1. GitHub 更新英雄案例和演示。
2. V2EX 或掘金二选一发布，不同时铺开。
3. 24 小时内集中回复并记录用户原话。
4. 邀请 3 位目标用户用真实任务试跑。
5. 只修最早、最高频的转化阻塞，再决定第二个渠道。

## 发布前检查

- 文案只保留一个主 CTA：拿真实任务试做第一份文档。
- 所有能力与当前仓库一致；不承诺 DOCX/PPTX、在线编辑器或尚未实现的集成。
- 所有数字都能链接到公开原始数据；否则删除数字。
- 外链使用渠道独立 UTM，帖子 URL 录入台账。
- 明确披露项目作者身份，不请求点赞或机械跨平台复制。
- Show HN 文本必须由项目作者本人撰写；HN 官方禁止发布生成式或 AI 编辑文本。

## 首轮完成标准

- 至少 10 位目标用户收到个性化试用邀请。
- 至少 5 位成功生成自己的文档。
- 收集至少 10 条具体阻塞或反对意见。
- 找到漏斗中损失最大的一个环节，并让下个月只围绕它改进。
